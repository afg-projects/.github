# 安全策略

## 🔒 支持范围

AFG 当前处于活跃开发阶段。安全修复优先覆盖各核心仓库的主分支。

## 📢 如何报告漏洞

**请不要把安全漏洞直接公开发到 Issue 中。**

### 报告方式

1. **GitHub 私密漏洞报告**（推荐）
   - 访问对应仓库的 Security 标签页
   - 点击 "Report a vulnerability"

2. **私下联系**
   - 通过 GitHub 私信联系维护者

### 报告中请包含

| 内容 | 说明 |
|------|------|
| 漏洞描述 | 清晰描述漏洞类型和影响 |
| 受影响仓库 | 指明哪个仓库受影响 |
| 复现步骤 | 提供详细的复现方法 |
| 影响评估 | 潜在的安全风险 |
| 缓解建议 | 如有可能，提供临时解决方案 |

## ⚡ 处理流程

```
收到报告 → 确认收到 → 复现验证 → 评估严重程度 → 制定修复方案 → 发布修复
```

我们会：

1. 尽快确认收到报告
2. 复现并评估严重程度
3. 协调修复
4. 在合适时机发布修复说明

## 🛡️ 安全最佳实践

使用 AFG Framework 时：

1. **依赖更新** - 定期更新依赖版本
2. **配置安全** - 不要在代码中硬编码敏感信息
3. **输入验证** - 始终验证用户输入
4. **权限控制** - 遵循最小权限原则
